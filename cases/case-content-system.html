Portfolio Content System

Role: UX writer and front-end builder

Timeline: 2 weeks

Goal: Publish and maintain case studies quickly with consistent narrative quality.

Problem

Portfolio updates were slow because every new project required deciding structure from scratch. As a result, promising work stayed unpublished and feedback loops happened too late.

Process

  • Defined a repeatable case framework: context, problem, process, outcomes, reflections
  • Built static page template and shared style system for consistent visual language
  • Added navigation map so every page is reachable in one or two clicks
  • Wrote editing checklist to ensure each case has measurable outcomes

Outcome

Time to publish a new case dropped from multiple days to a few focused hours. The site became easier for reviewers to scan, compare projects, and ask better interview questions.

What I would improve next

Add lightweight CMS-style data files and a build step while preserving static deployment simplicity.
