CSI 4160 Beacon

Repository: github.com/SaoodQureshi101/Beacon

Project: Real-time wireless signal processing pipeline with FastAPI, React, Flask, PostgreSQL, and MongoDB.

Architecture

The system combines a FastAPI streaming backend, two dashboards (React and Flask), and dual-database persistence. It supports both streaming and batch modes to compare operational behavior and performance.

Implemented Features

Real-time Processing Upgrade

  • WebSocket streaming endpoint: /ws/signal
  • Continuous signal ingestion and live inference
  • Live dashboard updates instead of batch-only workflows

Feature Engineering Layer

  • Signal amplitude variance
  • FFT frequency-domain features
  • Time-windowed signal segmentation

Simple Classification

  • Threshold-based classes: walking, standing, idle
  • Switchable mode support for basic ML model workflows
  • Training utility included for model updates

Dashboards and Observability

React dashboard: live signal graph, classification output, latency/throughput/accuracy metrics.

Flask dashboard: live canvas signal graph, classification output, and real-time metrics.

API, Storage, and Benchmarking

  • REST API for health, metrics, history, and mode switching
  • PostgreSQL persistence for structured records
  • MongoDB persistence for flexible document history
  • Batch vs streaming processing endpoints
  • Benchmark script for latency checks

Project Structure

  • backend/: FastAPI backend, persistence layer, benchmarking, model training
  • src/: signal pipeline and classifier
  • frontend/react-dashboard/: Vite + React dashboard
  • frontend/flask-dashboard/: Flask dashboard
  • docker-compose.yml: full-stack orchestration

Run with Docker (Recommended)

docker-compose up --build

  • FastAPI backend: http://localhost:8000
  • FastAPI docs: http://localhost:8000/docs
  • React dashboard: http://localhost:5173
  • Flask dashboard: http://localhost:5000
  • PostgreSQL: localhost:5432
  • MongoDB: localhost:27017

Core API Endpoints

  • GET /api/health
  • GET /api/metrics
  • GET /api/history?source=postgres|mongo&limit=300
  • POST /api/mode?new_mode=streaming|batch
  • POST /api/process-batch?size=256
  • WS /ws/signal

Local Setup (Without Docker)

  • pip install -r requirements.txt
  • uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
  • python frontend/flask-dashboard/app.py

React local:

  • cd frontend/react-dashboard
  • npm install
  • npm run dev

Utilities

  • Train model: python backend/train_model.py
  • Run benchmark: python backend/benchmark.py
